THE RULEনিয়ম

When Should You Create a Skill?কখন স্কিল তৈরি করা উচিত?

Any repeated multi-step procedure worth capturing once and reusing forever. যেকোনো বারবার করা বহু-ধাপের পদ্ধতি যা একবার ধরে রাখা এবং বারবার ব্যবহার করার মতো।
✓ Pre-release checks✓ প্রি-রিলিজ চেক ✓ Code review checklists✓ কোড রিভিউ চেকলিস্ট ✓ Changelog generation✓ চেঞ্জলগ তৈরি ✓ Incident runbooks✓ ইনসিডেন্ট রানবুক ✓ Onboarding walkthroughs✓ অনবোর্ডিং ওয়াকথ্রু ✓ Deploy verifications✓ ডিপ্লয় যাচাই
Golden signal: if you've done it by hand more than twice and typed the same instructions to Claude more than once — it's skill material.সোনালী সংকেত: যদি দুইবারের বেশি হাতে করেছ এবং একবারের বেশি Claude-কে একই নির্দেশনা টাইপ করেছ — এটা স্কিলের উপাদান।

Body — The Instructionsবডি — নির্দেশনা

Numbered Markdown steps. Be explicit: what to check, which tools to use, what the expected output looks like.ক্রমিক Markdown ধাপ। স্পষ্ট হও: কী চেক করতে হবে, কোন টুল ব্যবহার করতে হবে, প্রত্যাশিত আউটপুট কেমন হবে।
## Steps

1. Run `python manage.py showmigrations --list` and collect
   every unapplied migration.

2. For each unapplied migration, open the file and verify:
   - No column drop without a safe default
   - No table rename without an alias period
   - All new NOT NULL columns have a DB-level default

3. Execute `python manage.py migrate --run-syncdb --dry-run`
   and confirm exit code 0.

4. Run `python manage.py migrate --fake-initial --dry-run`
   to simulate rollback path. Confirm no errors.

5. Output a sign-off block:
   ```
   Migration Check — PASS/FAIL
   Checked: <date>
   Unapplied: <N>
   Risk: LOW | MEDIUM | HIGH
   Notes: …
   ```

LIVE BUILD · PART 2লাইভ বিল্ড · অংশ ২

SKILL.md — BodySKILL.md — বডি

## Steps

1. Find the last git tag:
   `git describe --tags --abbrev=0`

2. List all merge commits since that tag:
   `git log <tag>..HEAD --merges --oneline`

3. For each merge commit, extract the PR title and number.
   Group by conventional-commit prefix:
   - `feat:` → ✨ Features
   - `fix:`  → 🐛 Bug Fixes
   - `docs:` → 📝 Documentation
   - `chore:` / other → 🔧 Maintenance

4. Write a Markdown CHANGELOG block in this format:
   ```
   ## [vX.Y.Z] — YYYY-MM-DD
   ### ✨ Features
   - PR #123 — Add payment retry logic
   ### 🐛 Bug Fixes
   - PR #118 — Fix null pointer in OrderService
   ### 🔧 Maintenance
   - PR #115 — Bump dependencies
   ```

5. Ask: "Should I append this to CHANGELOG.md?" and
   wait for confirmation before writing to disk.

When Things Go Wrongযখন কিছু ঠিক না হয়

Problem: doesn't triggerসমস্যা: ট্রিগার হয় না
Fix: Sharpen the description. Add more natural-language variants of the trigger phrase. Think of how a new teammate would phrase the request. সমাধান: বিবরণ তীক্ষ্ণ করো। ট্রিগার বাক্যের আরও স্বাভাবিক রূপ যোগ করো। নতুন একজন টিমমেট কীভাবে অনুরোধ করতো তা ভাবো।
Problem: Claude wandersসমস্যা: Claude ঘুরে বেড়ায়
Fix: Tighten the body. Replace open-ended steps with precise commands. Add the exact output format you expect — Claude mirrors structure when you show it one. সমাধান: বডি আঁটো করো। অস্পষ্ট ধাপ নির্দিষ্ট কমান্ড দিয়ে প্রতিস্থাপন করো। তুমি যে আউটপুট ফর্ম্যাট চাও তা হুবহু দাও — তুমি দেখালে Claude কাঠামো অনুসরণ করে।
Problem: inconsistent outputসমস্যা: অসামঞ্জস্যপূর্ণ আউটপুট
Fix: Add an ## Example Output section at the end of the body. One concrete example eliminates most ambiguity. সমাধান: বডির শেষে ## Example Output বিভাগ যোগ করো। একটা কংক্রিট উদাহরণ বেশিরভাগ অস্পষ্টতা দূর করে।
A skill is a living document. Version 1 is a draft — expect to iterate 2–3 times before it's fully reliable.একটি স্কিল একটি জীবন্ত ডকুমেন্ট। সংস্করণ ১ একটি খসড়া — সম্পূর্ণ নির্ভরযোগ্য হওয়ার আগে ২-৩ বার পুনরাবৃত্তির প্রত্যাশা রাখো।

The Four Common Mistakesচারটি সাধারণ ভুল

  • ✗
    Generic descriptionঅস্পষ্ট বিবরণ
    "Checks stuff." Claude will never auto-trigger this. Fill it with actual phrases. "জিনিস চেক করে।" Claude কখনো স্বয়ংক্রিয়ভাবে এটা ট্রিগার করবে না। আসল বাক্য দিয়ে পূরণ করো।
  • ✗
    Body too long / no structureবডি অনেক লম্বা / কোনো কাঠামো নেই
    A wall of prose. Number your steps. Claude executes ordered lists reliably; paragraphs, less so. গদ্যের দেয়াল। ধাপে নম্বর দাও। Claude ক্রমিক তালিকা নির্ভরযোগ্যভাবে চালায়; অনুচ্ছেদ কম।
  • ✗
    No example outputকোনো উদাহরণ আউটপুট নেই
    Without a template, Claude invents format. Show what "done" looks like — copy-pasteable output block. টেমপ্লেট ছাড়া Claude নিজেই ফর্ম্যাট তৈরি করে। "শেষ" কেমন দেখায় তা দেখাও — কপি-পেস্টযোগ্য আউটপুট ব্লক।
  • ⚠️
    Wrong folderভুল ফোল্ডার
    Skills must live in .claude/skills/<name>/SKILL.md. Drop it anywhere else and Claude won't find it. স্কিল অবশ্যই .claude/skills/<name>/SKILL.md-তে থাকতে হবে। অন্য কোথাও রাখলে Claude খুঁজে পাবে না।
