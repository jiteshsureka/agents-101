WHAT DOESN'T BELONGযা রাখা উচিত নয়

Keep CLAUDE.md LeanCLAUDE.md হালকা রাখো

  • 🔑
    Secrets and API keys. Never. They belong in .env files and secret managers — not in a committed Markdown file.সিক্রেট ও API কি। কখনো না। সেগুলো .env ফাইল ও সিক্রেট ম্যানেজারে — কমিট করা Markdown ফাইলে নয়।
  • 📂
    Giant file dumps. Don't paste 500 lines of schema. Link to the schema file instead: See: src/Data/schema.sqlবিশাল ফাইল পেস্ট। ৫০০ লাইনের স্কিমা পেস্ট করবেন না। বরং লিঙ্ক করুন: See: src/Data/schema.sql
  • 🎯
    Per-task instructions. "Today we're building the banner widget" belongs in the prompt, not in the shared rulebook.প্রতি-টাস্কের নির্দেশনা। "আজ আমরা banner widget বানাচ্ছি" — এটা প্রম্পটে যাবে, শেয়ার্ড রুলবুকে নয়।
  • 📝
    Anything that changes per-run. Feature flags, ticket numbers, environment URLs, current sprint goals — these are dynamic, not structural.প্রতি-রানে পরিবর্তিত জিনিস। ফিচার ফ্ল্যাগ, টিকেট নম্বর, পরিবেশ URL, বর্তমান স্প্রিন্ট লক্ষ্য — এগুলো গতিশীল, কাঠামোগত নয়।
Rule of thumb: if it's true today AND will still be true six months from now, it belongs in CLAUDE.md. Otherwise, put it in the prompt.থাম্বের নিয়ম: আজ সত্য এবং ছয় মাস পরেও সত্য থাকবে — তাহলে CLAUDE.md-এ। নইলে প্রম্পটে।

Conventions, Architecture & DO NOTকনভেনশন, আর্কিটেকচার ও করবেন না

## Conventions
- Repository pattern for all data access (IRepository<T> from Nop.Core)
- snake_case for ALL database column names (EF column attribute required)
- XML doc comments on every public method and class
- No raw SQL — use LINQ on IQueryable<T>
- Services go in src/Services, controllers in src/Controllers (plugin MVC)

## Architecture
src/
  Core/       — domain models, IRepository contracts, service interfaces
  Data/       — EF mappings, migrations, repository implementations
  Services/   — business logic (stateless, injected via DI)
  Controllers/ — thin; delegates everything to Services
  Views/      — Razor; no business logic in .cshtml
See full diagram: docs/architecture.md

## DO NOT
- Never query the production connection string directly
- Never add jQuery — nopCommerce uses vanilla JS + Ajax helpers
- Never disable or bypass nopCommerce permission checks
- Never commit appsettings.json with real credentials

Three Ways Teams Get This Wrongটিমগুলো যে তিন উপায়ে ভুল করে

🕸️
Stale Instructionsপুরনো নির্দেশনা
You migrated from EF 6 to EF Core six months ago but CLAUDE.md still says "use ObjectContext." Claude follows it — confidently — and creates the wrong migration. Treat CLAUDE.md like code: review it when the architecture changes.ছয় মাস আগে EF 6 থেকে EF Core-এ মাইগ্রেট করেছ কিন্তু CLAUDE.md এখনও বলে "ObjectContext ব্যবহার করো।" Claude আত্মবিশ্বাসের সাথে মানে — এবং ভুল migration তৈরি করে। CLAUDE.md-কে কোডের মতো দেখাও: আর্কিটেকচার পরিবর্তন হলে রিভিউ করো।
⚔️
Contradictory Rulesপরস্পরবিরোধী নিয়ম
"Use async everywhere" in ## Conventions and "never make service methods async" in ## DO NOT. Claude picks one arbitrarily and is half-wrong either way. Audit for conflicts before committing.## Conventions-এ "সর্বত্র async ব্যবহার করো" এবং ## DO NOT-এ "service method কখনো async করবেন না।" Claude যেকোনো একটি বেছে নেয় এবং যেভাবেই হোক অর্ধেক ভুল। কমিটের আগে দ্বন্দ্ব পরীক্ষা করো।
🤯
Over-Stuffingঅতিরিক্ত ভরা
Adding every edge-case decision, every historical context note, every "we tried this once and it failed" anecdote. A 1,200-line CLAUDE.md buries the critical rules under noise. Claude weights all tokens equally — your most important rule competes with trivia.প্রতিটি এজ-কেস সিদ্ধান্ত, প্রতিটি ঐতিহাসিক প্রসঙ্গ, প্রতিটি "একবার চেষ্টা করেছিলাম এবং ব্যর্থ হয়েছিলাম" গল্প যোগ করা। ১,২০০ লাইনের CLAUDE.md শব্দের নিচে গুরুত্বপূর্ণ নিয়ম চাপা দেয়। Claude সব টোকেন সমানভাবে ওজন দেয় — তোমার সবচেয়ে গুরুত্বপূর্ণ নিয়ম তুচ্ছ জিনিসের সাথে প্রতিযোগিতা করে।
🔒
Forgetting to Commit Itকমিট করতে ভুলে যাওয়া
Mouneeta writes a great CLAUDE.md but never runs git add CLAUDE.md. Rafid's sessions have no context. The team loses the shared rulebook advantage entirely. Add CLAUDE.md to your repo init checklist.Mouneeta দুর্দান্ত CLAUDE.md লেখে কিন্তু কখনো git add CLAUDE.md রান করে না। Rafid-এর সেশনে কোনো কনটেক্সট নেই। টিম শেয়ার্ড রুলবুকের সুবিধা সম্পূর্ণ হারায়। রেপো init চেকলিস্টে CLAUDE.md যোগ করো।

DIAGNOSE FIRSTআগে নির্ণয় করো
Run /status in Claude Code to confirm CLAUDE.md loaded. Check for a conflicting user-level instruction. FIX ITঠিক করো
Make the rule more explicit: don't say "snake_case" — say "ALL database column names MUST use snake_case enforced via [Column("column_name")] attribute." KEY TAKEAWAYSমূল শিক্ষা
1. CLAUDE.md auto-loads every session — no prompt needed
2. Four levels: Enterprise → User → Project → Subdirectory
3. Memory ≠ CLAUDE.md — they serve different purposes
4. Token tax: every line costs on every run — stay lean
5. Commit it to git — or the team can't benefit from it ১. CLAUDE.md প্রতিটি সেশনে স্বয়ংক্রিয়ভাবে লোড — কোনো প্রম্পট দরকার নেই
