THE CONCEPTধারণা

What Is a Skill?স্কিল কী?

A reusable procedure card
you hand the apprentice once. একটি পুনর্ব্যবহারযোগ্য পদ্ধতি কার্ড
যা তুমি শিক্ষানবিশকে একবার দাও।
BEFORE SKILLSস্কিলের আগে
Rafid types "nop-station conventions" at the start of every session — project structure, test runner, PR style, style guide… রাফিদ প্রতিটি সেশনের শুরুতে "nop-station কনভেনশন" টাইপ করে — প্রজেক্ট স্ট্রাকচার, টেস্ট রানার, PR স্টাইল, স্টাইল গাইড…
AFTER SKILLSস্কিলের পরে
Rafid types /nop-conventions — or Claude auto-loads it when the request matches. Zero paste, zero re-explaining. রাফিদ /nop-conventions টাইপ করে — অথবা Claude রিকোয়েস্ট মিলে গেলে নিজেই লোড করে। শূন্য পেস্ট, শূন্য পুনরায় ব্যাখ্যা।
A skill is a folder with a SKILL.md inside. That file tells Claude exactly how to execute a named procedure — on demand or automatically.একটি স্কিল হল একটি ফোল্ডার যার ভেতরে একটি SKILL.md আছে। সেই ফাইলটি Claude-কে বলে কীভাবে একটি নামকৃত পদ্ধতি চালাতে হবে — চাহিদায় বা স্বয়ংক্রিয়ভাবে।

# new-episode skill
(everything below here is the body — loaded only when invoked)

Creates a new episode HTML file for agents-101.

## Usage

/new-episode

Prompts for season number, episode number, slug, title, subtitle.

## Steps

1. Copy template.html → seasons/season-N/ep-N-slug.html
2. Replace every placeholder with the provided values
3. Report the new file path to the user
Two parts: the frontmatter (name + description) is always visible to Claude. The body (steps, examples, notes) loads only when the skill is invoked. That split is the whole secret.দুটি অংশ: ফ্রন্টম্যাটার (নাম + বিবরণ) Claude-এর কাছে সবসময় দৃশ্যমান। বডি (ধাপ, উদাহরণ, নোট) শুধুমাত্র স্কিল ডাকা হলে লোড হয়। সেই বিভাজনটাই পুরো রহস্য।

PROGRESSIVE DISCLOSUREক্রমিক প্রকাশ

Three Loading Tiersতিনটি লোডিং স্তর

ALWAYSসবসময়
name + description — a one-liner listing in Claude's context. Costs almost nothing. Every session, every skill. নাম + বিবরণ — Claude-এর কনটেক্সটে একটি এক-লাইন তালিকা। প্রায় কোনো খরচ নেই। প্রতিটি সেশনে, প্রতিটি স্কিলের।
ON INVOKEডাকলে
full SKILL.md body — the steps, examples, notes. Loaded the moment Claude decides this skill applies. সম্পূর্ণ SKILL.md বডি — ধাপ, উদাহরণ, নোট। মুহূর্তেই লোড হয় যখন Claude সিদ্ধান্ত নেয় এই স্কিল প্রযোজ্য।
ON DEMANDপ্রয়োজনে
bundled files — scripts, templates, examples stored next to SKILL.md. Claude reads them only when the skill's steps say to. বান্ডেল করা ফাইল — SKILL.md-এর পাশে সংরক্ষিত স্ক্রিপ্ট, টেমপ্লেট, উদাহরণ। স্কিলের ধাপ বললেই Claude সেগুলো পড়ে।
This is why skills don't bloat your context window even when you have 30 of them. Only the triggered skill's body costs real tokens.এই কারণেই ৩০টি স্কিল থাকলেও স্কিলগুলো তোমার কনটেক্সট উইন্ডো ফুলিয়ে তোলে না। শুধুমাত্র চালু হওয়া স্কিলের বডিই আসল টোকেন খরচ করে।

Creates a new episode HTML file for agents-101.

/new-episode

2. Replace all {{PLACEHOLDERS}} with provided values
3. Report the new file path

Three Ways Skills Go Wrongস্কিল তিনভাবে ব্যর্থ হয়

VAGUE DESCRIPTIONঅস্পষ্ট বিবরণ
"Helps with coding tasks" — matches almost everything. The skill either fires when it shouldn't, or Claude can't distinguish it from three others and ignores all of them. "কোডিং কাজে সাহায্য করে" — প্রায় সব কিছুর সাথে মেলে। স্কিল হয় ভুল সময়ে চালু হয়, অথবা Claude তিনটির মধ্যে পার্থক্য করতে পারে না এবং সবগুলো উপেক্ষা করে।
BLOATED BODYফোলা বডি
A SKILL.md with 500 lines of context dumps the entire team wiki into one skill. Once invoked, it eats your whole context window and slows everything down. Move large assets to bundled files instead. ৫০০ লাইনের SKILL.md পুরো টিম উইকি একটি স্কিলে ঢেলে দেয়। ইনভোকেশনের পরে, এটা তোমার পুরো কনটেক্সট উইন্ডো খেয়ে ফেলে এবং সব ধীর করে দেয়। বড় অ্যাসেটগুলো বান্ডেল ফাইলে সরিয়ে দাও।
ONE SKILL = MANY JOBSএক স্কিল = অনেক কাজ
A "do-everything" skill with 12 sub-commands is hard to trigger reliably and hard to maintain. One skill should do one job, with a sharp, specific description. ১২টি সাব-কমান্ড সহ "সব-করো" স্কিল নির্ভরযোগ্যভাবে ট্রিগার করা এবং রক্ষণাবেক্ষণ করা কঠিন। এক স্কিল এক কাজ করবে, একটি তীক্ষ্ণ, নির্দিষ্ট বিবরণ সহ।
