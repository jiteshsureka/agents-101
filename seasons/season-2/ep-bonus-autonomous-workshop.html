HOOKS · THE KEY NEW PIECE হুক · মূল নতুন অংশ

The harness runs hooks. Not Claude. হার্নেস হুক চালায়। Claude না।

// .claude/settings.json — hook configuration
{
  "hooks": {
    "SessionStart": [
      {
        "hooks": [{
          "type": "command",
          "command": "cat CLAUDE.md project-context.md"
        }]
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [{

      }
    ]
  }
}

The harness executes these shell commands — Claude never decides whether to skip them. That's what makes hooks reliable. হার্নেস এই শেল কমান্ড চালায় — Claude কখনো সেগুলো এড়িয়ে যাওয়ার সিদ্ধান্ত নিতে পারে না। এটাই হুককে নির্ভরযোগ্য করে।

// .claude/settings.json — PR auto-review hook
{
  "hooks": {
    "PostToolUse": [
      {
        // Fires whenever Claude writes or edits any source file
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            // Run in headless mode; --max-turns caps cost
            "command": "claude -p '/code-review' --max-turns 30"
          }
        ]
      }
    ]
  }
}

The harness sees the Write tool complete, then immediately shells out to run the skill — no prompt from the developer needed. হার্নেস Write tool সম্পন্ন হতে দেখে, তারপর স্বয়ংক্রিয়ভাবে শেল করে স্কিল চালায় — ডেভেলপারের prompt দরকার নেই।

LIVE BUILD · PR REVIEW PIPELINE · STEP 2 লাইভ বিল্ড · PR রিভিউ পাইপলাইন · ধাপ ২

The skill spawns one reviewer per changed file. স্কিল প্রতিটা পরিবর্তিত ফাইলে একজন রিভিউয়ার spawn করে। 

# .claude/skills/code-review/index.md  (skill prompt body)

## What to do
1. Run `git diff --name-only HEAD` to get changed files.
2. For each file, spawn a subagent with this prompt:
   "Review {{file}} for correctness, security, and style.
    Output: file path, severity (critical/warn/info), finding, fix."
3. Collect all subagent outputs.
4. Aggregate into one Markdown report grouped by severity.
5. Print the report and exit.

## Constraints
- Spawn at most 10 subagents concurrently (rate-limit guard).
- Each subagent gets --max-turns 15.
- If zero files changed, print "nothing to review" and exit 0.

The skill is plain Markdown — readable, editable, version-controlled. No code to maintain. The Agent tool does the spawning. স্কিল সাধারণ Markdown — পাঠযোগ্য, সম্পাদনযোগ্য, version-controlled। রক্ষণাবেক্ষণের কোড নেই।

COST & SAFETY · AT AUTONOMOUS SCALE খরচ এবং নিরাপত্তা · স্বায়ত্তশাসিত স্কেলে

Power demands guardrails. শক্তির জন্য গার্ডরেল দরকার।

  • 💰 Spend caps — set token or dollar limits per session and per skill run. Stop runaway cost before it compounds. খরচ সীমা — প্রতি সেশন ও স্কিল রানে টোকেন বা ডলার সীমা নির্ধারণ করো।
  • 🔢 Bounded subagent counts — cap concurrent spawns (--max-turns, -P 10). Never unbounded. সীমাবদ্ধ সাবএজেন্ট সংখ্যা — concurrent spawn সীমাবদ্ধ করো। কখনো অসীম না।
  • 🚦 Approval gates — for risky actions (deploys, DB writes), require a human y/n before the hook chain proceeds. অনুমোদন গেট — ঝুঁকিপূর্ণ কাজে hook chain চালু হওয়ার আগে মানুষের অনুমোদন নাও।
  • 🔭 Observability — log every hook event, every subagent spawn, every cost delta. Invisible work is unmanageable. (S1E6 callback) পর্যবেক্ষণযোগ্যতা — প্রতিটা হুক ইভেন্ট, spawn, খরচ পরিবর্তন লগ করো।

Autonomy without observability is a black box. Observability without caps is a blank cheque. You need both. পর্যবেক্ষণ ছাড়া স্বায়ত্তশাসন একটা কালো বাক্স। সীমা ছাড়া পর্যবেক্ষণ একটা ফাঁকা চেক। দুটোই দরকার।

WATCH OUT · GOTCHAS সতর্ক থাকো · ফাঁদসমূহ

Autonomy bites back when you're not careful. অসাবধান হলে স্বায়ত্তশাসন কামড়ায়।

  • 🔄 Runaway loops — a PostToolUse hook that itself triggers a Write can recurse infinitely. Always check: does this hook produce output that could re-trigger itself? লুপ রানওয়ে — PostToolUse হুক যদি নিজেই Write ট্রিগার করে, তাহলে অসীম পুনরাবৃত্তি হয়।
  • 💸 Unbounded subagent spawning — a skill that loops over files without a concurrency cap can exhaust API rate limits and budget in minutes. অসীম সাবএজেন্ট spawn — concurrency cap ছাড়া ফাইলের উপর loop করা স্কিল মিনিটে বাজেট শেষ করতে পারে।
  • 🧱 Hooks that block real work — a slow hook (e.g. full test suite on every keystroke) turns automation into a drag. Match hook cost to event frequency. হুক যা কাজ আটকায় — ধীর হুক (প্রতিটা কীস্ট্রোকে পূর্ণ টেস্ট) automation-কে বোঝায় পরিণত করে।
  • 🧑‍⚖️ No human checkpoint on risky actions — autonomous is not the same as unchecked. Delete, deploy, and send always need a gate. ঝুঁকিপূর্ণ কাজে কোনো মানবিক চেকপয়েন্ট নেই — স্বায়ত্তশাসন মানে অনিয়ন্ত্রিত না।
DESIGN PRINCIPLES ডিজাইন নীতি

Build the workshop to last. কর্মশালা দীর্ঘস্থায়ী করো।

Smallest possible triggerসম্ভব ক্ষুদ্রতম ট্রিগার

Match hooks to specific tool matchers ("Write|Edit" not "*"). Surgical beats broad.হুক নির্দিষ্ট tool matcher-এ match করো। সার্জিক্যাল সেরা।

Skills are specs, not scriptsস্কিল স্পেসিফিকেশন, স্ক্রিপ্ট না

Write skills in plain language. Claude reads them as instructions, not code to execute. Easier to maintain and update.স্কিল সাধারণ ভাষায় লেখো। Claude সেগুলো নির্দেশ হিসেবে পড়ে।

Human in the loop at the endশেষে মানুষ লুপে থাকে

The goal isn't to remove humans — it's to remove toil. A good autonomous workshop surfaces a clear decision, not raw output.লক্ষ্য মানুষ সরানো না — কষ্টকর কাজ সরানো।

THE BIG QUESTION বড় প্রশ্ন

"How much of your daily craft can be
turned into a skill, distributed to subagents,
and triggered by a hook —
leaving only the judgement for you?" "তোমার দৈনন্দিন কাজের কতটুকু
স্কিলে পরিণত করা যায়, সাবএজেন্টে বিতরণ করা যায়,
এবং হুক দিয়ে ট্রিগার করা যায় —
শুধু বিচার-বুদ্ধি তোমার জন্য রেখে?"

That's the design challenge of the autonomous workshop. It's never "automate everything" — it's "automate the routine so humans focus on the irreplaceable." এটাই স্বায়ত্তশাসিত কর্মশালার ডিজাইন চ্যালেঞ্জ। লক্ষ্য "সব স্বয়ংক্রিয়" না — "রুটিন স্বয়ংক্রিয় করো যাতে মানুষ অপরিহার্যে মনোযোগ দেয়।"
