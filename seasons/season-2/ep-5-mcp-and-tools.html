DECISION POINTসিদ্ধান্তের বিন্দু

When to Wrap an API as an MCP Toolকখন API-কে MCP টুল হিসেবে র‍্যাপ করবেন

WRAP IT WHEN…র‍্যাপ করো যখন…
  • ✓ You need it more than once a week✓ সপ্তাহে একবারের বেশি দরকার
  • ✓ It's a live system (not a static doc)✓ এটা লাইভ সিস্টেম (স্ট্যাটিক ডক নয়)
  • ✓ Multiple team members will use it✓ একাধিক টিম সদস্য ব্যবহার করবে
  • ✓ You want to compose it inside skills✓ স্কিলের ভেতরে কম্পোজ করতে চাও
SKIP IT WHEN…এড়িয়ে যাও যখন…
  • ✗ One-off fetch you'll never repeat✗ একবারের ফেচ, আর কখনো নয়
  • ✗ Public URL Claude can just WebFetch✗ পাবলিক URL Claude সরাসরি WebFetch করতে পারে
  • ✗ Data fits in a file you can commit✗ ডেটা একটা ফাইলে ধরে যায় যা কমিট করা যায়
  • ✗ No auth needed, no rate limits✗ কোনো অথ নেই, রেট লিমিট নেই
Rule of thumb: if you'd reach for a real API client in code, you should probably wrap it as an MCP tool.সহজ নিয়ম: কোডে যদি রিয়েল API ক্লায়েন্ট ব্যবহার করতেন, সম্ভবত MCP টুল হিসেবে র‍্যাপ করা উচিত।

Skills Orchestrate MCP Toolsস্কিলস MCP টুলস অর্কেস্ট্রেট করে

A skill's body is plain instructions. It can reference MCP tool names directly — Claude will call them at runtime.একটা স্কিলের বডি সরল নির্দেশনা। এটা সরাসরি MCP টুলের নাম রেফার করতে পারে — Claude রানটাইমে সেগুলো কল করবে।
## skill: sprint-status ## Fetches live sprint data and posts a summary to Slack. 1. Use mcp__jira__get_sprint_issues with project=NOPST and sprint=active to fetch all open tickets. 2. Group tickets by status: To Do / In Progress / Done. 3. Format a standup summary: - Count per status, list blockers (tickets with label=blocked). - Mention the top 3 In-Progress tickets with assignee. 4. Use mcp__slack__send_message channel=#dev-standup to post the formatted summary. 5. Reply to the user with a confirmation + ticket count.
The skill body never hardcodes data — it commands Claude to fetch it live at run time. This is what makes skills dynamic, not static.স্কিল বডি কখনো ডেটা হার্ডকোড করে না — এটা Claude-কে লাইভে ফেচ করতে নির্দেশ দেয়। এটাই স্কিলকে ডায়নামিক করে।

// settings.json { "permissions": { "allow": [ "mcp__jira__get_sprint_issues", "mcp__jira__get_issue", "mcp__slack__send_message" ], "deny": [ "mcp__jira__delete_issue", "mcp__jira__transition_issue" ] } }
APPROVAL GATESঅ্যাপ্রুভাল গেট
Some tool calls require human sign-off before Claude proceeds. Claude surfaces a confirmation prompt — no action taken until you type yes. কিছু টুল কলের জন্য Claude এগোনোর আগে মানুষের অনুমোদন দরকার। Claude একটি কনফার্মেশন প্রম্পট দেখায় — আপনি yes না লেখা পর্যন্ত কোনো কাজ হয় না।
💡 Callback to S1·E6: the same enterprise safety controls — budget limits, allow-lists, audit logs — apply to MCP tools too. 💡 S1·E6-এর কথা মনে করুন: একই এন্টারপ্রাইজ সেফটি কন্ট্রোল — বাজেট লিমিট, অ্যালাউ-লিস্ট, অডিট লগ — MCP টুলেও প্রযোজ্য।
Principle: grant the minimum tool surface your skill actually needs. A sprint-report skill never needs to delete tickets.নীতি: স্কিলের আসলে যা দরকার তার ন্যূনতম টুল সারফেস দাও। স্প্রিন্ট-রিপোর্ট স্কিলের কখনো টিকেট ডিলিট করার দরকার নেই।

GOTCHASসতর্কতা

MCP Pitfalls to AvoidMCP-এর ভুলগুলো এড়ানো

  • 🔑
    Secrets in the skill bodyস্কিল বডিতে সিক্রেট
    Never write API tokens or passwords in prompt.md. Use env in settings.json — and never commit settings.json with real values to git. prompt.md-তে কখনো API টোকেন বা পাসওয়ার্ড লিখবেন না। settings.json-এ env ব্যবহার করুন — এবং সত্যিকার মান দিয়ে settings.json কখনো git-এ কমিট করবেন না।
  • 🌐
    Over-broad tool permissionsঅতিরিক্ত টুল পারমিশন
    Allowing mcp__jira__* gives Claude write access to every Jira action. Be explicit: allow only the tools the skill actually calls. mcp__jira__* অ্যালাউ করলে Claude সব Jira অ্যাকশনে রাইট অ্যাক্সেস পায়। স্পষ্ট হোন: শুধু স্কিল যা কল করে সে টুলগুলোই অ্যালাউ করুন।
  • 📦
    Unverified third-party MCP serversযাচাইবিহীন থার্ড-পার্টি MCP সার্ভার
    A malicious MCP server can exfiltrate data or execute destructive actions. Only install servers from trusted sources; review the source before running. একটি দুর্ভাবনামূলক MCP সার্ভার ডেটা চুরি করতে বা ক্ষতিকর কাজ চালাতে পারে। শুধুমাত্র বিশ্বস্ত সোর্স থেকে সার্ভার ইনস্টল করুন; চালানোর আগে সোর্স রিভিউ করুন।
  • ⏱️
    No rate-limit awarenessরেট-লিমিট সচেতনতার অভাব
    Claude won't automatically throttle tool calls. A looping skill can hit Jira's rate limit in seconds. Add retry logic or a delay step in the skill body. Claude স্বয়ংক্রিয়ভাবে টুল কল থ্রটল করবে না। লুপিং স্কিল সেকেন্ডে Jira-র রেট লিমিটে পৌঁছাতে পারে। স্কিল বডিতে রিট্রি লজিক বা ডিলে স্টেপ যোগ করুন।

Building Your Own MCP Serverনিজের MCP সার্ভার তৈরি করা

If no community server covers your in-house API, build one in TypeScript with the official SDK — it takes less than 50 lines for a basic read-only tool.যদি কোনো কমিউনিটি সার্ভার আপনার ইন-হাউস API কভার না করে, অফিশিয়াল SDK দিয়ে TypeScript-এ একটা তৈরি করুন — বেসিক রিড-অনলি টুলের জন্য ৫০ লাইনের কম লাগে।
// my-api-mcp/index.ts (TypeScript, official SDK) import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js"; import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js"; import { z } from "zod"; const server = new McpServer({ name: "my-api", version: "1.0.0" }); server.tool( "get_open_orders", { limit: z.number().default(20) }, async ({ limit }) => { const data = await fetchOrders({ status: "open", limit }); return { content: [{ type: "text", text: JSON.stringify(data) }] }; } ); const transport = new StdioServerTransport(); await server.connect(transport);
Register it in settings.json just like any other server. Your entire team can then call mcp__my-api__get_open_orders from any skill.settings.json-এ অন্য সার্ভারের মতোই রেজিস্টার করুন। তারপর পুরো টিম যেকোনো স্কিল থেকে mcp__my-api__get_open_orders কল করতে পারবে।
