Writing the plugin-scaffold Skillplugin-scaffold স্কিল লেখা

# .claude/skills/plugin-scaffold/SKILL.md ## Trigger /plugin-scaffold ## Args $PLUGIN_NAME # e.g. NopStation.Plugin.Payments.StripeConnect $AUTHOR # e.g. nop-station $VERSION # e.g. 4.80.0 ## Task Generate a valid nopCommerce plugin folder structure at src/Plugins/$PLUGIN_NAME/ containing: 1. Description.txt — SystemName, FriendlyName, Author, Version, SupportedVersions 2. $PLUGIN_NAME.csproj — targets net8.0, refs Nop.Core + Nop.Services 3. Plugin.cs — class implementing BasePlugin, IPlugin · InstallAsync / UninstallAsync with localisation resource cleanup 4. Infrastructure/RouteProvider.cs — implements IRouteProvider · Order = 1, empty RegisterRoutes body 5. Controllers/ — placeholder controller inheriting BasePluginController ## Constraints - Use NopCommerce 4.80 API surface only - Async/await throughout; NO synchronous DB calls - Follow naming in CLAUDE.md section "Plugin conventions" - Do NOT generate migration files — those come from /migration-check

# CLAUDE.md — nop-station nopCommerce projects ## Stack - ASP.NET Core 8 · Razor Pages + MVC areas - Entity Framework Core 8 · Code-first migrations - Nop.Core / Nop.Services / Nop.Web.Framework - xUnit + Moq for tests · GitHub Actions CI ## Build & test dotnet build src/NopStation.sln dotnet test src/Tests/ --logger trx ## Plugin conventions - SystemName format: NopStation.Plugin.{Area}.{Name} - Always implement BasePlugin; area-specific interface second - DB access via IRepository<T> — NEVER use DbContext directly - Service classes go in Services/; keep controllers thin - Localization via ILocalizationService — no hardcoded strings ## DO NOT - Synchronous IO — always async/await - Raw SQL strings — use LINQ or IRepository - Skip InstallAsync resource registration - Modify core Nop.* source files - Add <Nullable>disable</Nullable> to .csproj

GOTCHASসতর্কতা

Three Ways to Shoot Yourselfনিজেকে ক্ষতি করার তিনটি উপায়

  • ⚡
    Over-automating judgment callsবিচার-নির্ভর সিদ্ধান্তকে অতিরিক্ত স্বয়ংক্রিয় করা
    Some decisions — "is this API breaking?" — need a senior eye, not a skill. Skills work for repeatable structure, not nuanced judgment. কিছু সিদ্ধান্ত — "এটা কি API ভাঙছে?" — একজন সিনিয়রের চোখ দরকার, স্কিল নয়। স্কিলগুলো পুনরাবৃত্ত কাঠামোর জন্য কাজ করে, সূক্ষ্ম বিচারের জন্য নয়।
  • 📢
    Skills that encode bad patternsখারাপ প্যাটার্ন এনকোড করা স্কিলস
    If your old code does synchronous DB calls, a scaffold skill will too — and generate that mistake at scale. Fix the pattern first, then codify it. পুরানো কোড যদি সিঙ্ক্রোনাস DB কল করে, স্ক্যাফোল্ড স্কিলও তাই করবে — এবং সেই ভুলটি স্কেলে তৈরি করবে। আগে প্যাটার্ন ঠিক করুন, তারপর কোডিফাই করুন।
  • 🔄
    Stale skills on framework upgradeফ্রেমওয়ার্ক আপগ্রেডে পুরনো স্কিলস
    nopCommerce 4.80 → 4.90 changed several plugin interfaces.
