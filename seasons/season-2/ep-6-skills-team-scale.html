Plugins: Bundle Skills + Config into One Packageপ্লাগইন: স্কিল + কনফিগ এক প্যাকেজে

A plugin bundles skills, subagents, hooks, and MCP server config — installable across many repos with a single command.একটি প্লাগইন স্কিল, সাবএজেন্ট, হুক এবং MCP সার্ভার কনফিগ একসাথে বান্ডেল করে — একটি কমান্ডে অনেক রেপোতে ইনস্টল করা যায়।
nopstation-plugin/           # sharable plugin package
├── package.json             # name, version, description
└── .claude/
    ├── skills/
    │   ├── review-pr/
    │   │   └── skill.md
    │   └── seed-db/
    │       └── skill.md
    ├── agents/
    │   └── qa-agent/
    │       └── agent.md
    ├── hooks/
    │   └── pre-commit-lint.sh
    └── settings.json           # MCP server config, permissions
One plugin = all the team's shared automation in one versioned, installable artifact.একটি প্লাগইন = একটি ভার্সনড, ইনস্টলযোগ্য আর্টিফ্যাক্টে দলের সমস্ত শেয়ার্ড অটোমেশন।

Distribute & Enable the Pluginপ্লাগইন বিতরণ ও এনেবল করো

Publish to your org's registry (or a private npm), then install in any repo:তোমার অর্গের রেজিস্ট্রিতে (বা একটি প্রাইভেট npm-এ) প্রকাশ করো, তারপর যেকোনো রেপোতে ইনস্টল করো:
# 1. Publish the plugin (from plugin repo)
npm publish --access restricted

# 2. In any project repo — install the plugin
claude plugin install @nopstation/claude-plugin

# 3. Enable it for this project
claude plugin enable @nopstation/claude-plugin

# 4. Mouneeta opens Claude Code — skills are live
/ns-review-pr       # ← works immediately
/ns-seed-db         # ← works immediately
/ns-changelog       # ← works immediately

Pitfalls When Sharing Skills at Scaleস্কেলে স্কিল শেয়ার করার সময় বিপদ

  • ⚠️
    Drift: A dev tweaks their personal copy of a skill. Now personal and project versions diverge silently. Fix: treat the repo copy as the source of truth, always.ড্রিফট: একজন ডেভ স্কিলের ব্যক্তিগত কপি পরিবর্তন করে। এখন ব্যক্তিগত এবং প্রজেক্ট সংস্করণ নীরবে আলাদা হয়ে যায়। সমাধান: রেপো কপিকে সর্বদা সত্যের উৎস হিসেবে বিবেচনা করো।
  • ⚠️
    Unreviewed skills: A shortcut skill lands in main without a PR. It has bad instructions or prompt-injects something unsafe. The Three-Step Guild Rolloutতিন-ধাপের গিল্ড রোলআউট

STEP 1 — COLLECTধাপ ১ — সংগ্রহ
Identify the best personal skills across the team. Nominate them for the shared repo via PR. দল জুড়ে সেরা ব্যক্তিগত স্কিল সনাক্ত করো। PR-এর মাধ্যমে শেয়ার্ড রেপোর জন্য মনোনীত করো।
STEP 2 — REVIEWধাপ ২ — রিভিউ
At least two reviewers check: accuracy, safety, no secrets, clear instructions, named owner. কমপক্ষে দুজন রিভিউয়ার চেক করে: নির্ভুলতা, নিরাপত্তা, কোনো সিক্রেট নেই, স্পষ্ট নির্দেশনা, নামকৃত মালিক।
STEP 3 — DISTRIBUTEধাপ ৩ — বিতরণ
Merge to main. Plugin auto-updates. Every dev gets the improvement on next git pull. main-এ মার্জ করো। প্লাগইন স্বয়ংক্রিয়ভাবে আপডেট হয়। পরের git pull-এ প্রতিটি ডেভ উন্নতি পায়।
Three steps from "Rafid's personal skill" to "everyone on the team has it." Takes about 15 minutes per skill."রাফিদের ব্যক্তিগত স্কিল" থেকে "দলের সবাই এটা পেয়েছে" পর্যন্ত তিনটি ধাপ। প্রতিটি স্কিলে প্রায় ১৫ মিনিট সময় লাগে।
