ISOLATION MODESআইসোলেশন মোড

Worktree vs Shared DirectoryWorktree বনাম শেয়ার্ড ডিরেক্টরি

isolation: "worktree"
Each subagent gets its own git worktree — a separate directory on its own branch. Safe for write operations. Agents can't step on each other's files. প্রতিটো সাবএজেন্ট নিজের git worktree পায় — আলাদা ব্র্যাঞ্চে আলাদা ডিরেক্টরি। রাইট অপারেশনের জন্য নিরাপদ। এজেন্টরা একে অপরের ফাইল স্পর্শ করতে পারে না।
USE FOR: write tasksব্যবহার করো: রাইট কাজে
Default (no isolation)ডিফল্ট (কোনো আইসোলেশন নেই)
All agents share the same working directory. Fine for read-only tasks (analysis, auditing, report generation). Risky if agents write to disk. সব এজেন্ট একই ওয়ার্কিং ডিরেক্টরি শেয়ার করে। শুধু পড়ার কাজে ঠিক আছে (বিশ্লেষণ, অডিট, রিপোর্ট)। ডিস্কে লিখলে ঝুঁকিপূর্ণ।
USE FOR: read-only tasksব্যবহার করো: শুধু পড়ার কাজে
Habib's security audit is read-only — agents just read code and report findings. Default isolation is fine. No worktrees needed.হাবিবের সিকিউরিটি অডিট শুধু পড়া — এজেন্টরা কোড পড়ে ফলাফল জানায়। ডিফল্ট আইসোলেশনই যথেষ্ট। Worktree দরকার নেই।

What an Agent Call Looks LikeAgent কল কেমন দেখায়

# Claude Code's Agent tool — the fields you control

Agent({
  description: "Security audit — src/auth/ module",

  prompt: """
    Read all files in src/auth/. Look for:
    - SQL/NoSQL injection vulnerabilities
    - Tokens or secrets stored in plain text
    - Session handling bugs (fixation, no expiry)
    - Overly permissive CORS or cookie settings

    Return a markdown list of findings. If none found, say so.
    Do NOT modify any files. Do NOT spawn subagents.
  """,

  # optional — picks a specialized agent type
  subagent_type: "security-reviewer"
})
That's the full surface area. Description (for logging), prompt (the task), optional type. The agent runs and returns one message.এটাই পুরো ইন্টারফেস। বিবরণ (লগিংয়ের জন্য), প্রম্পট (কাজ), ঐচ্ছিক টাইপ। এজেন্ট চলে এবং একটি বার্তা ফেরায়।
WHEN TO USEকখন ব্যবহার করবে

The Subagent Fitness Checkসাবএজেন্ট উপযুক্ততা পরীক্ষা

  • ✓ Batch analysis — same task across N targets (files, modules, PRs) ব্যাচ বিশ্লেষণ — N লক্ষ্যে একই কাজ (ফাইল, মডিউল, PR)
  • ✓ Parallel read-only work — no shared writes between agents সমান্তরাল পড়ার কাজ — এজেন্টদের মধ্যে শেয়ার্ড রাইট নেই
  • ✓ Well-defined deliverable — you know exactly what "done" looks like সুনির্দিষ্ট ফলাফল — "সম্পন্ন" কেমন দেখায় তা জানো
  • ✓ Each subtask takes > 5 min — worth the overhead of spawning প্রতিটো উপকাজ > ৫ মিনিট — স্পন করার ওভারহেড সার্থক
  • ✓ No mid-flight steering needed — you trust the prompt to guide it মাঝপথে স্টিয়ারিং দরকার নেই — প্রম্পটই পথ দেখাবে
Pass all five → reach for subagents. Fail any one → use multiple sessions instead (Ep 2).পাঁচটিতেই পাস → সাবএজেন্ট ব্যবহার করো। যেকোনো একটিতে ফেল → পরিবর্তে একাধিক সেশন (Ep 2)।
WHEN NOT TO USEকখন ব্যবহার করবে না

Subagents Are the Wrong Tool When…সাবএজেন্ট ভুল টুল যখন…

  • ✗ The task needs live steering — you'll want to course-correct mid-run কাজে লাইভ স্টিয়ারিং দরকার — মাঝপথে দিকনির্দেশনা বদলাতে হবে
  • ✗ Scope is unknown / exploratory — you're not sure what "done" means yet স্কোপ অজানা / অন্বেষণমূলক — "সম্পন্ন" এখনো স্পষ্ট নয়
  • ✗ Agents write to the same files — race conditions, corrupted output এজেন্টরা একই ফাইলে লেখে — রেস কন্ডিশন, দূষিত আউটপুট
  • ✗ Very short tasks (< 2 min each) — overhead of spawning > benefit খুব ছোট কাজ (< ২ মিনিট) — স্পনের ওভারহেড > সুবিধা

THE CRAFTদক্ষতা

Writing the Orchestrator Promptঅর্কেস্ট্রেটর প্রম্পট লেখা

The orchestrator prompt has one job: decompose the task and spawn N subagents. Structure it clearly.অর্কেস্ট্রেটর প্রম্পটের একটাই কাজ: কাজ ভাগ করো এবং N সাবএজেন্ট স্পন করো। স্পষ্টভাবে কাঠামো দাও।
1 · CONTEXT১ · প্রসঙ্গ
What the codebase is. What kind of audit. Which modules to cover.কোডবেস কী। কী ধরনের অডিট। কোন মডিউল কভার করতে হবে।
2 · SUBTASK LIST২ · উপকাজের তালিকা
Spawn Agent for src/auth/, Agent for src/payments/, Agent for src/api/, Agent for src/admin/.src/auth/-এর জন্য Agent, src/payments/-এর জন্য Agent, src/api/-এর জন্য Agent, src/admin/-এর জন্য Agent স্পন করো।
3 · OUTPUT FORMAT৩ · আউটপুট ফরম্যাট
Each agent returns a structured markdown block. After all return, compile into a single security report.প্রতিটো এজেন্ট স্ট্রাকচার্ড মার্কডাউন ব্লক ফেরায়। সব ফেরার পর একক সিকিউরিটি রিপোর্ট সংকলন করো।
The output format specification is critical. If you don't define it, each agent invents its own structure and synthesis becomes painful.আউটপুট ফরম্যাট স্পেসিফিকেশন গুরুত্বপূর্ণ। না দিলে প্রতিটো এজেন্ট নিজের কাঠামো বানায় এবং সংকলন কঠিন হয়।
THE CRAFTদক্ষতা

The Subagent Prompt Patternসাবএজেন্ট প্রম্পট প্যাটার্ন

SCOPEস্কোপ
Exactly which files/dirs to read. Nothing outside this.ঠিক কোন ফাইল/ডিরেক্টরি পড়তে হবে। এর বাইরে কিছু না।
DELIVERABLEফলাফল
What to return. Exact format: markdown findings, severity labels, file:line references.কী ফেরাতে হবে। ঠিক ফরম্যাট: মার্কডাউন ফাইন্ডিং, সিভিয়ারিটি লেবেল, file:line রেফারেন্স।
CONSTRAINTSসীমাবদ্ধতা
Read-only. Do NOT write files. Do NOT spawn subagents. Do NOT run bash commands.শুধু পড়া। ফাইল লিখবে না। সাবএজেন্ট স্পন করবে না। bash কমান্ড চালাবে না।
EMPTY CASEখালি ক্ষেত্র
If no issues found: return "No findings in [module]." Always return something.কোনো সমস্যা না পেলে: "[মডিউল]-এ কোনো ফাইন্ডিং নেই।" সবসময় কিছু ফেরাও।
The "Do NOT spawn subagents" line is not optional. Every subagent prompt must have it. Without it, costs spiral."সাবএজেন্ট স্পন করবে না" লাইনটি ঐচ্ছিক নয়। প্রতিটো সাবএজেন্ট প্রম্পটে থাকতে হবে। ছাড়া হলে খরচ বাড়ে।

COSTখরচ

Budget Before You Spawnস্পন করার আগে বাজেট করো

THE MATHহিসাব
N subagents × (input tokens + output tokens) each.

Habib: 4 agents × ~40k tokens each = ~160k tokens total.

Estimate before spawning — not after. N সাবএজেন্ট × (ইনপুট + আউটপুট টোকেন) প্রতিটো।

হাবিব: ৪ এজেন্ট × ~৪০k টোকেন = ~১৬০k টোকেন মোট।

স্পনের আগে অনুমান করো — পরে নয়।
COST CONTROLSখরচ নিয়ন্ত্রণ
  • • Tight scope = fewer tokens• টাইট স্কোপ = কম টোকেন
  • • Explicit output format = less rambling• স্পষ্ট আউটপুট ফরম্যাট = কম বাড়তি কথা
  • • Start with 2 agents, then scale• ২ এজেন্ট দিয়ে শুরু, তারপর বাড়াও
  • • Check /cost after each run• প্রতিটো রানের পর /cost চেক করো
Subagents multiply token usage. That's the trade-off. The time saving usually justifies it — but you should know the number before you commit.সাবএজেন্ট টোকেন ব্যবহার বাড়ায়। এটাই ট্রেড-অফ। সময় সাশ্রয় সাধারণত সার্থক হয় — কিন্তু কমিট করার আগে সংখ্যা জানা উচিত।

You Can't Steer Mid-Flightমাঝপথে স্টিয়ার করা যায় না

THE PROBLEMসমস্যা
A subagent goes off-track. You won't know until it returns — possibly 20 minutes later with a useless result. No interrupt. No course-correct mid-run. একটি সাবএজেন্ট পথ হারিয়ে ফেলে। তুমি জানতে পারবে না যতক্ষণ না সে ফেরে — হয়তো ২০ মিনিট পরে অপ্রয়োজনীয় ফলাফল নিয়ে। কোনো বাধা নেই। মাঝপথে কোনো সংশোধন নেই।
THE MITIGATIONSপ্রতিকার
  • • Tight scope — reduce surface area• টাইট স্কোপ — পৃষ্ঠ এলাকা কমাও
  • • Explicit output contract• স্পষ্ট আউটপুট চুক্তি
  • • Test with 1 agent before N• N-এর আগে ১ এজেন্টে পরীক্ষা করো
  • • Validate output structure on return• ফেরার পর আউটপুট কাঠামো যাচাই করো
If you need to course-correct mid-task, use sessions (Ep 2). If you trust the prompt to handle it, use subagents (this episode).মাঝপথে সংশোধন দরকার হলে, সেশন ব্যবহার করো (Ep 2)। প্রম্পটকে বিশ্বাস করলে, সাবএজেন্ট ব্যবহার করো (এই এপিসোড)।
