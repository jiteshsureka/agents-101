THE PRIMITIVEমূল টুল

Git Worktrees — The Essential ToolGit Worktree — অপরিহার্য টুল

A worktree is a second (or third) checkout of your repo into a separate directory — each on its own branch. No stashing. No checkout switching. Multiple branches active simultaneously on disk.ওয়ার্কট্রি হলো তোমার রেপোকে একটি আলাদা ডিরেক্টরিতে দ্বিতীয় (বা তৃতীয়) চেকআউট — প্রতিটি নিজস্ব ব্র্যাঞ্চে। কোনো স্ট্যাশিং নেই। কোনো চেকআউট সুইচিং নেই। একসাথে ডিস্কে একাধিক ব্র্যাঞ্চ সক্রিয়।
WITHOUT WORKTREESওয়ার্কট্রি ছাড়া
One working dir. Must stash changes to switch branches. Can't run two Claude sessions on two branches simultaneously. একটি ওয়ার্কিং ডিরেক্টরি। ব্র্যাঞ্চ সুইচ করতে পরিবর্তন স্ট্যাশ করতে হবে। একসাথে দুটি ব্র্যাঞ্চে দুটো Claude সেশন চালানো যাবে না।
WITH WORKTREESওয়ার্কট্রি সহ
Three directories. Three branches. Three Claude sessions. All live simultaneously. No stash needed. তিনটি ডিরেক্টরি। তিনটি ব্র্যাঞ্চ। তিনটো Claude সেশন। সবকিছু একসাথে চলছে। কোনো স্ট্যাশ দরকার নেই।

Worktree Commandsওয়ার্কট্রি কমান্ড

# Create three isolated worktrees (run from main repo dir)
git worktree add ../nopstation-email-val    feature/email-validation
git worktree add ../nopstation-order-tests  feature/order-tests

Each path is now a fully functional repo checkout on its own branch. Open a terminal in each one and launch Claude.প্রতিটি পাথ এখন নিজের ব্র্যাঞ্চে একটি সম্পূর্ণ কার্যকরী রেপো চেকআউট। প্রতিটিতে একটি টার্মিনাল খোলো এবং Claude চালু করো।
SELECTIONনির্বাচন

The Parallelizability Checkসমান্তরালতা যোগ্যতা পরীক্ষা

Run every ticket through these four questions before you assign it to a parallel session:সমান্তরাল সেশনে নিয়োগের আগে প্রতিটি টিকেটকে এই চারটি প্রশ্নের মধ্য দিয়ে চালাও:
  • ✓ Independent files — no overlap with other parallel tickets? স্বাধীন ফাইল — অন্য সমান্তরাল টিকেটের সাথে ওভারল্যাপ নেই?
  • ✓ No shared state — different DB tables, different config, different modules? কোনো শেয়ার্ড স্টেট নেই — আলাদা DB টেবিল, আলাদা কনফিগ, আলাদা মডিউল?
  • ✓ No ticket-to-ticket dependency — neither task needs the other's output? টিকেট-থেকে-টিকেট নির্ভরতা নেই — কোনো কাজ অপরের আউটপুট চায় না?
  • ✓ Each task will take more than 15 minutes of Claude time? প্রতিটি কাজ Claude-এর ১৫ মিনিটের বেশি সময় নেবে?
Pass all four → safe to run in parallel. Fail any one → resolve the dependency first, then parallelize.চারটিতেই পাস → সমান্তরালে নিরাপদ। যেকোনো একটিতে ফেল → প্রথমে নির্ভরতা সমাধান করো, তারপর সমান্তরাল করো।

THE CRAFTদক্ষতা

Write a Focused Launch Promptফোকাসড লঞ্চ প্রম্পট লিখো

A lazy prompt wastes tokens and goes off-track. A focused prompt is the most important thing you write per session.অস্পষ্ট প্রম্পট টোকেন নষ্ট করে এবং পথ হারায়। ফোকাসড প্রম্পট হলো প্রতি সেশনে তোমার লেখা সবচেয়ে গুরুত্বপূর্ণ জিনিস।
Context:  nop-station checkout flow. React + TypeScript. Zod for validation.

Task:     Add email validation to the address form in checkout.
          - Validate on blur and on form submit
          - Show inline error: "Please enter a valid email address"
          - Reuse existing Zod schema: src/lib/validators.ts → emailSchema

Files:    src/components/checkout/AddressForm.tsx (primary)
          src/lib/validators.ts (read-only — do not modify)

Pattern: Context → Task → Files → Do NOT. Every session should start with this structure.প্যাটার্ন: Context → Task → Files → Do NOT। প্রতিটি সেশন এই কাঠামো দিয়ে শুরু করা উচিত।

THE CHECKপরীক্ষা

60 Seconds to Assess a Session৬০ সেকেন্ডে একটি সেশন মূল্যায়ন

  • 1. Scan the last 10 lines of output — is it moving in the right direction? আউটপুটের শেষ ১০ লাইন স্ক্যান করো — এটা কি সঠিক দিকে যাচ্ছে?
  • 2. Run a quick git diff --stat — does the file set match the task? দ্রুত git diff --stat চালাও — ফাইল সেট কি কাজের সাথে মেলে?
  • 3. Check for a blocking question — if Claude is waiting on you, answer it. ব্লকিং প্রশ্ন চেক করো — Claude যদি তোমার জন্য অপেক্ষা করছে, উত্তর দাও।
  • 4. If it's on track — leave it running and move to the next session. যদি সঠিক পথে থাকে — চলতে দাও এবং পরের সেশনে যাও।
Most rotations are 60 seconds. You're not reviewing the code — you're checking the heading. Deep review happens at PR time.বেশিরভাগ রোটেশন ৬০ সেকেন্ড। তুমি কোড রিভিউ করছ না — দিক পরীক্ষা করছ। গভীর রিভিউ PR সময়ে হবে।

The Context Refresh Promptকনটেক্সট রিফ্রেশ প্রম্পট

When you return to a session after 20 minutes, don't assume it remembers where it was. Ground it fast.২০ মিনিট পরে কোনো সেশনে ফিরে এলে, ধরে নিও না সে মনে আছে কোথায় ছিল। দ্রুত ভিত্তি দাও।
# The context refresh pattern — paste this when you return to a session

Review the diff so far.
Current task:  Add email validation to CheckoutForm.tsx
Status:       You implemented blur validation. Submit validation is still missing.
Next step:    Add validation on form submit — same error message as blur.
Don't touch:  src/lib/validators.ts (read-only)
This works because it re-anchors the session without repeating the full context. Fast. Precise. Prevents drift.এটা কাজ করে কারণ পূর্ণ কনটেক্সট পুনরাবৃত্তি না করে সেশনকে পুনরায় স্থির করে। দ্রুত। নির্ভুল। ড্রিফট প্রতিরোধ করে।
WATCH OUTসাবধান

When a Session Gets Stuckযখন সেশন আটকে যায়

SIGNSলক্ষণ
  • • Asking a clarifying question• স্পষ্টীকরণের প্রশ্ন করছে
  • • Going in the wrong direction• ভুল দিকে যাচ্ছে
  • • Modifying files it shouldn't• যে ফাইল স্পর্শ করা উচিত নয় তা পরিবর্তন করছে
  • • Loops on the same step• একই ধাপে লুপ করছে
  • • Long output, wrong result• দীর্ঘ আউটপুট, ভুল ফলাফল
FIXESসমাধান
  • • Answer the question → continue• প্রশ্নের উত্তর দাও → চালিয়ে যাও
  • • Paste the error + "fix this only"• এরর পেস্ট করো + "শুধু এটা ঠিক করো"
  • • "Stop. Revert [file]. Focus on [X]."• "থামো। [ফাইল] রিভার্ট করো। [X]-এ ফোকাস করো।"
  • • Hard reset: new prompt with tight scope• হার্ড রিসেট: শক্ত স্কোপ সহ নতুন প্রম্পট
  • • Park the session, handle later• সেশন পার্ক করো, পরে সামলাও
Most stalls take 30 seconds to fix. The mistake is micromanaging healthy sessions instead of fixing the stuck ones.বেশিরভাগ আটকে যাওয়া ৩০ সেকেন্ডে সমাধান হয়। ভুলটা হলো সুস্থ সেশনগুলোকে মাইক্রোম্যানেজ করা, আটকে যাওয়াগুলো ঠিক করার বদলে।
EDGE CASEবিশেষ ক্ষেত্র

Mid-Flight: Unexpected Dependencyমাঝপথে: অপ্রত্যাশিত নির্ভরতা

SHAZZAD · HOUR 1 "Session B discovered it needs a validateOrder() helper that Session A is also building. They'll conflict." "Session B আবিষ্কার করেছে তার একটি validateOrder() হেল্পার দরকার যা Session A-ও তৈরি করছে। তারা কনফ্লিক্ট করবে।"
OPTION 1 · PARKবিকল্প ১ · পার্ক
