GREEN LIGHTসবুজ বাতি

Tasks That Run Well in Parallelযে কাজগুলো সমান্তরালে ভালো চলে

  • ✓ Independent features — no shared code between them স্বতন্ত্র ফিচার — তাদের মধ্যে কোনো শেয়ার্ড কোড নেই
  • ✓ Tests for code you finished yesterday গতকাল শেষ করা কোডের টেস্ট
  • ✓ API docs, changelogs, or release notes API ডকস, চেঞ্জলগ, বা রিলিজ নোট
  • ✓ Work across separate microservices or modules আলাদা মাইক্রোসার্ভিস বা মডিউলে কাজ
  • ✓ Isolated refactors with clean module boundaries পরিষ্কার মডিউল বাউন্ডারিসহ বিচ্ছিন্ন রিফ্যাক্টর
The test: if two tasks don't touch the same files and don't need each other's output — they can run in parallel.পরীক্ষা: দুটো কাজ যদি একই ফাইল স্পর্শ না করে এবং একে অপরের আউটপুট না চায় — সমান্তরালে চলতে পারে।
RED LIGHTলাল বাতি

Tasks That Don't Parallelizeযে কাজগুলো সমান্তরালে চলে না

  • ✗ Task B needs Task A's output before it can start Task B শুরু হওয়ার আগে Task A-এর আউটপুট দরকার
  • ✗ Both sessions would modify the same source file দুটো সেশনই একই সোর্স ফাইল পরিবর্তন করবে
  • ✗ Session B depends on Session A's running service Session B Session A-এর চলমান সার্ভিসের উপর নির্ভরশীল
  • ✗ Deeply coupled changes — one PR breaks the other গভীরভাবে জড়িত পরিবর্তন — একটা PR অন্যটা ভাঙে
The parallel check: "Could I hand these two tasks to different people right now?" If no — don't parallelize yet.সমান্তরাল চেক: "এই দুটো কাজ এখনই দুজন আলাদা মানুষকে দিতে পারি?" যদি না — এখন সমান্তরাল করবেন না।

WATCH OUTসাবধান

Parallel Isn't Freeসমান্তরাল বিনামূল্যে নয়

  • ⚠️
    Cost: 3 sessions ≈ 3× token usage. Focused prompts reduce waste.খরচ: ৩ সেশন ≈ ৩× টোকেন ব্যবহার। ফোকাসড প্রম্পট অপচয় কমায়।
  • ⚠️
    Context tax: Each session needs its own full context from scratch. Write it clearly.কনটেক্সট ট্যাক্স: প্রতিটি সেশনকে স্ক্র্যাচ থেকে নিজস্ব পূর্ণ কনটেক্সট দিতে হবে। স্পষ্টভাবে লিখো।
  • ⚠️
    Merge overhead: More PRs = more review cycles. Smaller PRs help compensate.মার্জ ওভারহেড: বেশি PR = বেশি রিভিউ চক্র। ছোট PR ক্ষতিপূরণ করে।
  • ⚠️
    Mental load: Context-switching sessions has a real cognitive cost.মানসিক চাপ: সেশন কনটেক্সট-সুইচিংয়ের বাস্তব জ্ঞানীয় খরচ আছে।
WHEN IT PAYSযখন লাভজনক

The Parallelism Payoff Testসমান্তরালতার মূল্য পরীক্ষা

WORTH IT WHEN…লাভজনক যখন…
  • ✓ Each task takes > 20 minutes✓ প্রতিটি কাজ > ২০ মিনিট সময় নেয়
  • ✓ Tasks are fully independent✓ কাজগুলো সম্পূর্ণ স্বাধীন
  • ✓ Merge is straightforward✓ মার্জ সহজ
  • ✓ You have 2+ tickets queued✓ ২+ টিকেট কিউতে আছে
  • ✓ Sessions don't share any files✓ সেশনগুলো কোনো ফাইল শেয়ার করে না
SKIP IT WHEN…এড়িয়ে যাও যখন…
  • ✗ Tasks are short (< 15 min)✗ কাজগুলো ছোট (< ১৫ মিনিট)
  • ✗ Deep dependency between tickets✗ টিকেটের মধ্যে গভীর নির্ভরতা
  • ✗ Only 1 ticket in the queue✗ কিউতে মাত্র ১টি টিকেট
  • ✗ Already in deep flow on one task✗ ইতিমধ্যে একটি কাজে গভীর ফ্লোতে আছো
  • ✗ Touching the same config/schema✗ একই কনফিগ/স্কিমা স্পর্শ করা হচ্ছে
Default trigger: start a new session whenever you notice yourself waiting on Claude. That moment is your signal.ডিফল্ট ট্রিগার: যখনই দেখবে Claude-এর জন্য অপেক্ষা করছ, নতুন সেশন শুরু করো। সেই মুহূর্তটাই তোমার সংকেত।
REAL TALKসত্যি কথা

Context-Switching Has a Costকনটেক্সট-সুইচিংয়ের একটা খরচ আছে
