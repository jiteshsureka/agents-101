Claude Code Skills

Pre-built slash-command skills your team can invoke. Each entry covers what it does, when to trigger it, and real enterprise use cases.

Code Quality

/code-review

Reviews the current diff for bugs, simplification opportunities, and architectural issues at configurable effort levels (low → ultra).

/code-review [low|medium|high|ultra]

Use before every PR. Add --fix to auto-apply findings, --comment to post inline PR comments.

Code Quality

/simplify

Scans recently changed code for reuse, simplification, and efficiency cleanups — then applies the fixes automatically.

/simplify

Run after a feature lands to tighten up the implementation before it's merged to main.

Testing

/tdd-guide

Enforces write-tests-first methodology. Generates failing tests before implementation, targets 80%+ behavioral coverage.

/tdd-guide

Enterprise teams use this on every new feature to prevent test debt accumulating over sprints.

Testing

/test-coverage

Analyzes test coverage gaps, identifies untested paths, and generates targeted tests for the most critical missing coverage.

/test-coverage

Run on legacy modules before a migration to establish a safety net.

Architecture

/plan

Designs a feature implementation plan: analyzes codebase patterns, produces file-level blueprints, data flow, and build order.

/plan

Use before starting any feature larger than a single function. Prevents mid-sprint architectural pivots.

Architecture

/feature-dev

Full feature development skill: plan → implement → test → review in a guided, checkpointed flow.

/feature-dev

Hand off a feature spec and let the agent run the entire cycle with approval gates at key milestones.

DevOps

/build-fix

Auto-resolves build failures, TypeScript errors, and linter issues with minimal surgical diffs. Gets the build green fast.

/build-fix

Trigger automatically via CI hook when a build fails to unblock teammates without context-switching.

DevOps

/pm2

Manages PM2 process lifecycle — start, restart, logs, monitoring — for Node.js services in dev and staging environments.

/pm2

Use in microservice environments where multiple processes need coordinated restart after deployments.

Planning

/prp-plan

Generates a Product Requirements Prompt (PRP) from a high-level description — structured enough for agents to implement reliably.

/prp-plan

Bridge the gap between product requirements and agent-executable specs. Use at sprint planning.

Planning

/prp-implement

Implements a PRP document — reads the spec, builds out the feature with tests, then posts a summary of what was done.

/prp-implement [prp-file]

Pair with /prp-plan for a fully automated spec-to-code pipeline on well-defined features.

Security

/security-review

Scans for OWASP Top 10, secrets in code, SSRF, injection, unsafe crypto, and authentication flaws in changed files.

/security-review

Mandatory before any PR that touches auth, API endpoints, user input, or data persistence layers.

Planning

/multi-plan

Plans a complex feature by decomposing it into parallel workstreams that multiple agents or sessions can execute concurrently.

/multi-plan

Use for large features where parts are independent — generates a dependency graph and session assignments.
